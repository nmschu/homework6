Homework 6 - Node.js

Node.js is a server side application. It is capable of non-blocking IO, which helps a lot with real-time web applications (like a chatroom. Check out the video below for the example). Node.js also has a very good error checking system, and encourages people to practice proper error checking.

Node.js is different than a lot of the more traditional server-side languages in a few ways. On the Node.js website, they say that some of the big concepts were influenced from Ruby and Python, but also it presents an event loop as a language construct, and not as a library that it is with those other languages. It also uses JavaScript which is widely known. Most people who have even a small amount of web development experience would be able to start working with Node.js.

Non-blocking IO means that it will not stop the system. As mentioned before, node.js is a non-blocking IO. Using Node.js as an example, when you are running something and there is a break in the code (like execute this, then wait 5 seconds, then execute this other thing), during the wait time the system is free to perform another action. Where traditionally, when you have a wait (under the same circumstances) the system will "freeze". This means no other process can be performed until the wait is finished. The whole concept of the non-blocking IO means that it never sleeps, rests, waits, or any other variation of the word.
